## Descripción del Proyecto

Este repositorio contiene un proyecto de portafolio que simula el trabajo real de un **Analista de Banca Digital** en una institución financiera. El objetivo es demostrar habilidades prácticas aplicables a cualquier banco o fintech:

- **SQL** (consultas complejas, CTEs, window functions)
- **Power BI** (modelado, DAX, visualizaciones)
- **Power Query** (limpieza y transformación de datos)
- **Python** (generación de datos, análisis exploratorio)
- **Excel Avanzado** (análisis complementario)
- **Análisis de Negocio** (insights y recomendaciones estratégicas)

## Objetivo del Análisis

Simular el análisis completo de la operación de banca digital de un banco, respondiendo preguntas clave de negocio que son universales en la industria financiera:

1. ¿Cómo es el perfil y comportamiento de los clientes digitales?
2. ¿Qué productos tienen mayor adopción y rentabilidad?
3. ¿Cómo evoluciona el uso de canales digitales (App vs Web)?
4. ¿Qué clientes tienen mayor riesgo de abandono (churn)?
5. ¿Qué insights podemos generar para mejorar la retención y venta cruzada?

## Tecnologías y Herramientas

| Herramienta | Uso en el Proyecto |
|------------|-------------------|
| **Python 3.9+** | Generación de datos sintéticos, análisis exploratorio (Pandas, NumPy, Matplotlib) |
| **SQL** | Consultas para extracción y análisis de datos (SQLite) |
| **Power BI** | Dashboard interactivo con medidas en DAX |

| **Excel** | Análisis complementario con tablas dinámicas |

## 📊 Datos Utilizados

Los datos son **sintéticos pero realistas**, generados específicamente para este proyecto con Python. Simulan el comportamiento típico de clientes en cualquier entidad financiera:

- **Clientes:** 5,000 registros con edad, género, ubicación, antigüedad y segmento
- **Productos:** Tenencia de cuentas, tarjetas, inversiones y créditos por cliente
- **Transacciones:** 100,000 movimientos con montos, tipos y canales
- **Uso Digital:** 200,000 eventos en App/Web (logins, acciones, tiempo de sesión)

*Los datos incluyen errores realistas (valores nulos, fechas inconsistentes, outliers) para simular un entorno de trabajo real.*
